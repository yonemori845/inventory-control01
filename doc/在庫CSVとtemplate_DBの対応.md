| 更新日 | 2026-04-05 |

## 結論

- **`.ods` をそのままアップロードすることはできません。** アプリが受け付けるのは、所定の**1行目ヘッダー付きの CSV**だけです。
- `template_DB.ods` は **「商品」「在庫」「注文」など複数シート**の設計用テンプレで、列名も意味も **在庫CSV用フォーマットと一致しません**。
- そのため、Calc / Excel で **列を対応させて1シートに並べ**、**UTF-8 の CSV** として保存し直す必要があります。

## アプリが要求する CSV（1行目は完全一致）

次の **1行目をそのままコピー**してください（列の順番・スペルは変更不可）。

```text
group_code,group_name,sku_code,jan_code,name_variant,color,size,quantity,reorder_point,safety_stock,unit_price_ex_tax,is_active,group_description,sort_order
```

テンプレート付きのダウンロードは、アプリの **在庫一覧 →「CSV ダウンロード」** から取得できます。

### 各列の意味

| 列名 | 必須 | 説明 |
|------|------|------|
| `group_code` | ○ | 親商品のコード。**同一グループの SKU 行では同じ値**にすると1つの親にまとまります。 |
| `group_name` | ○ | 親商品の表示名。 |
| `sku_code` | ○ | **SKU ごとに一意**のコード（テンプレの「商品ID」を使ってもよいし、`MLS-101-S` のようにしてもよい）。 |
| `jan_code` | ○ | JAN。**全体で一意**である必要があります。 |
| `name_variant` | 任意 | バリエーション名（空欄可）。 |
| `color` | 任意 | 色（空欄可）。 |
| `size` | ○推奨 | サイズ等。空にする場合は空文字。 |
| `quantity` | ○ | 現在庫（0 以上の整数）。 |
| `reorder_point` | ○ | 再発注点（整数）。 |
| `safety_stock` | ○ | 安全在庫（整数）。 |
| `unit_price_ex_tax` | ○ | 税抜単価（数値）。テンプレの「税抜単価」に相当。 |
| `is_active` | ○ | `true` / `false`（または `1` / `0`）。 |
| `group_description` | 任意 | 親の説明（空欄可）。 |
| `sort_order` | ○ | 親の並び（整数。よく分からなければ `0`）。 |

## `template_DB.ods` のシートとの対応（目安）

### 「商品」シート

| template_DB.ods（商品） | 在庫CSVの列 |
|-------------------------|-------------|
| 商品コード | `group_code`（同じ親でまとめたい場合） |
| 商品名 | `group_name` |
| サイズ | `size` |
| JANコード | `jan_code` |
| 税抜単価 | `unit_price_ex_tax` |
| 商品ID | `sku_code` にそのまま使うか、`{商品コード}-{サイズ}` など**一意**になるよう命名 |
| 画像URL | **在庫 CSV には含めない**（テンプレに列があってもインポートでは無視）。**商品画像**は Supabase Storage に置き、DB の `product_skus.image_path` を更新する（**同一親商品の SKU では同一パス**を推奨。`doc/商品画像_収集と利用方針.md`） |
| 税込単価 | 在庫CSVでは使わない（税抜のみ） |

同じ「商品コード」でサイズ違い（S/M）の行がある場合、**`group_code` と `group_name` を共通**にし、`sku_code` と `jan_code` を行ごとに変えるイメージです。

### 「在庫」シート

| template_DB.ods（在庫） | 在庫CSVの列 |
|-------------------------|-------------|
| 商品ID | 「商品」シートと突き合わせて、その行の `quantity` にする |
| 在庫数量 | `quantity` |

「商品ID」が在庫シートに無い SKU は、数量を `0` にするなど運用で決めてください。

### 「注文」「注文詳細」シート

在庫マスタの CSV 取込には**使いません**（注文データは別フェーズの画面／別データです）。

## 作業手順（Calc / Excel）

1. **在庫一覧で「CSV ダウンロード」**し、1行目をコピーする。
2. `template_DB.ods` の「商品」をベースに、上表のとおり値を**列として横に並べる**（VLOOKUP / XLOOKUP で「在庫」シートから数量を引くとよい）。
3. **`sku_code` が重複しないこと**、`jan_code` が重複しないことを確認する。
4. **名前を付けて保存**で形式を **「テキスト CSV」**、文字コード **UTF-8**（BOM 付きでも可）を選ぶ。
5. アプリの **「CSV アップロード」** でそのファイルを選ぶ。

## よくあるエラー

- **「ヘッダーがテンプレートと一致しません」**  
  → 1行目の英小文字・カンマ区切りがテンプレと**完全一致**しているか確認（日本語列名のままでは不可）。
- **`duplicate sku_code` / `jan_code`**  
  → 同一 CSV 内、または既にDBにある SKU/JAN とぶつかっていないか確認。
