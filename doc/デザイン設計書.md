本書は UI の**見た目・コンポーネント・トークン・画面レイアウト方針**を定義する。ブランド・数値は架空である。

---

## 1. 目的・対象外

### 1.1 目的

- shadcn/ui と Tailwind を前提に、**一貫した視覚言語**と**誤操作しにくい操作系**を定める。
- 機能設計書の **画面 ID** ごとに、主要ブロックとコンポーネント選択方針を示す。

### 1.2 対象外

- ロゴ・イラストのブランド制作（プレースホルダでよい）。
- アニメーションの詳細仕様（必要最低限のみ）。

### 1.3 実装との差分（メモ）

- **shadcn/ui**: 設計上は primitives の第一選択だが、現リポジトリでは **未導入**（Tailwind ＋自作コンポーネント）。導入時は §4.1 に沿って置き換えを進める。
- **モバイルナビ**: §5.1 の `Sheet` に代わり、**カスタムドロワー**（`AppShell`）で同等の UX を提供している。

## 2. デザイン原則

### 2.1 情報の優先順位（在庫・注文・売上）

1. **数値の異常**（在庫アラート、在庫不足エラー）を最優先で可視化する。
2. **確定系アクション**（注文確定、CSV 反映）の直前に、合計・影響範囲を明示する。
3. ダッシュボードでは **KPI → トレンド → ランキング → アラート** の順で視線を誘導する。

### 2.2 誤操作防止（確定・在庫不足・削除）

- **注文確定**：プライマリボタンは 1 画面 1 つに近づける。確定前に**税込合計**と**明細件数**を常時表示。
- **在庫不足**：ダイアログまたはインラインアラートで**どの SKU が足りないか**を列挙（機能設計 §8.2 と一致）。
- **破壊的操作**（将来の削除等）：確認ダイアログ＋ラベルを避ける色（例：destructive バリアント）。

### 2.3 アクセシビリティ方針

- インタラクティブ要素に**フォーカスリング**を残す（`outline` を無効化しすぎない）。
- コントラストは WCAG 2.1 AA を**目標**（厳密監査は課題範囲で任意）。
- フォームエラーは**テキスト**で説明する（色のみに依存しない）。

## 3. デザイントークン

### 3.1 色（ライト／ダーク方針）

- **デフォルト**：ライト基調。ダークモードは**任意**（shadcn の `dark` クラス戦略に従う）。
- **意味付け**：成功（入庫完了）、警告（在庫アラート）、エラー（確定失敗）、情報（ヘルプ）を**セマンティックトークン**（CSS 変数）経由で参照する。

### 3.2 タイポグラフィ

- 本文：**sans-serif**（Next.js 既定または Inter 等、プロジェクトで 1 つに固定）。
- 数値・金額：**タブラー数字**を使う場合は `font-variant-numeric: tabular-nums` を検討。
- 見出し階層：ページタイトル（h1）→ セクション（h2）→ カード内（h3）。

### 3.3 間隔・角丸・影

- 8px グリッドを基本（`p-4`, `gap-4` 等）。カード内は `p-4`〜`p-6`。
- 角丸：`rounded-lg` をカード・ダイアログの既定とする。
- 影：浮遊カードのみ `shadow-sm` 程度。過剰な影は避ける。

### 3.4 Tailwind／shadcn との対応

- 生の hex をコンポーネントに散らさず、`globals.css` の CSS 変数と Tailwind の `bg-primary` 等に寄せる。
- 新規 UI はまず **shadcn の primitives**（Button, Card, Table, Dialog, Sheet, Toast）で組む。

## 4. コンポーネント方針

### 4.1 shadcn/ui の採用範囲

| 用途 | コンポーネント（例） |
|------|----------------------|
| ナビ | `Sidebar` または `NavigationMenu` + モバイルは `Sheet` |
| 一覧 | `Table`、行展開で入れ子は `Collapsible` |
| フォーム | `Input`, `Label`, `Select`, `Checkbox` |
| 確定・警告 | `Dialog`, `AlertDialog` |
| 通知 | `Sonner` または `Toast` |
| ローディング | `Skeleton` |

### 4.2 フォーム・バリデーション表示

- フィールド直下にエラーテキスト。サーバーエラーはトースト＋該当フィールドに反映可能なら反映。

### 4.3 データテーブル・入れ子一覧

- **親行**：商品名、SKU 数、合計在庫（集計）、アラート件数バッジ。
- **子行**：SKU、JAN、色・サイズ、在庫、単価、操作（編集／スキャンへの導線）。
- モバイル：テーブルが窮屈な場合は**カードスタック**に切替（同一データ）。

### 4.4 ダイアログ・トースト

- 在庫不足・CSV 失敗は **AlertDialog** または **Toast**（重要度に応じて使い分け）。
- トーストは同一エラーの連打を抑制（任意）。

### 4.5 バーコード／カメラ UI

- **大きな「スキャン開始」ボタン**と、カメラプレビュー領域（アスペクト比 4:3 程度）。
- 連続スキャン（入庫）モードはトグルで明示。誤スキャン時は**直前 1 件の取り消し**を任意で付けてよい（要件外だが UX 向上）。

## 5. レスポンシブ

### 5.1 ブレークポイント

| 名称 | 幅（Tailwind 目安） |
|------|---------------------|
| sm | 640px |
| md | 768px |
| lg | 1024px |
| xl | 1280px |

- **ナビ**：`lg` 未満はハンバーガー＋`Sheet`。

### 5.2 モバイルでの主要フロー

- **入庫スキャン**・**注文作成のスキャン**は片手操作を意識し、主要ボタンを親指圏に配置。
- ダッシュボードのチャートは横スクロールまたは縦積み。

## 6. 画面別 UI 仕様

> 画面 ID は `機能設計書.md` §3 と同一。

### 6.1 SCR-AUTH-LOGIN

- 中央カード：Google ボタン、メール／パスワードフォーム、サインアップリンク（提供する場合）。
- 背景は単色または薄いパターン（パフォーマンス優先）。

### 6.2 SCR-DASH

- 上部：**期間セレクタ**（デフォルト：当月など）。
- KPI カード 4 点群、次に折れ線／棒チャート、次に TOP10 テーブル、最下部に在庫アラートリスト（`Badge` で警告度）。

### 6.3 SCR-INV-LIST

- サマリ行（グループ数・総 SKU・総在庫・アラート数）。
- 検索バー（名前／コード／JAN）。
- アクション：**CSV ダウンロード**、**CSV アップロード**、**入庫スキャン**（モーダルまたは同一ページセクション）。
- 本体：入れ子テーブル。在庫編集は **インライン** または **Sheet**。
- 確定系に相当するボタン（保存・入庫確定・CSV アップロード等）は **白背景＋枠線**で統一（真っ黒の塗りつぶしは使わない）。
- 親行バッジは子 SKU 行数を **「N SKU」** と表示。
- 子行の **SKU コード** は **SCR-INV-SKU**（`/inventory/sku/[skuId]`）へリンク。
- バリエーション列では、テキストと在庫アラートバッジの間に余白を取る。

### 6.4 SCR-INV-HIST

- フィルタ：期間、SKU、理由区分。
- テーブル：日時、SKU、JAN、理由、増減、実行者。
- **SKU** 列は **SCR-INV-SKU** へリンク。

### 6.5 SCR-ORD-LIST

- サマリカード（総注文・本日・平均など）。
- テーブル：注文番号、日時、税込合計。行クリックで SCR-ORD-DET へ。

### 6.6 SCR-ORD-NEW

- 左（または上）：スキャン／SKU 追加エリア。
- 右（または下）：明細テーブル（数量 ±、行削除）、フッタに税抜・税・税込、**確定**プライマリ。
- 在庫不足時：モーダルで不足リスト。

### 6.7 SCR-ORD-DET

- ヘッダカード：注文番号、日時、金額内訳。
- 明細テーブル：SKU、数量、単価、行小計、税の内訳列（表示方針は機能設計の税ルールに従う）。

### 6.8 SCR-RPT

- フィルタパネル＋チャート群＋表。**エクスポート**はドロップダウン（PDF／Excel）。

### 6.9 SCR-SET

- セクション分割：アカウント、接続状態（ステータスバッジ）、権限表示、**開発のみ**シード（`Environment` バッジ付きで区別）。

### 6.10 SCR-ADM-USERS

- ユーザーテーブル：メール、表示名、ロール、`Select` で変更＋保存。

## 7. PWA・オフライン時

### 7.1 インストール促進 UI

- 初回訪問後、バナーまたは設定から「ホームに追加」を案内（ブラウザ API に合わせる）。

### 7.2 オフライン時のシェル・メッセージ

- **シェル＋静的アセットのみ**オフライン表示（要件定義 §7.2）。
- オフライン時はメインコンテンツ域に「接続を確認してください」を表示し、**データの読み書きは試みない**。
