1. 作るもの（概要）
--------------------------------------------------------------------------------

次の機能を持つ Web アプリケーションを想定している。

【横断・基盤】
  - ログイン機能
  - JAN コード・バーコードスキャン
  - Supabase によるデータベース連携
  - GitHub Actions による CI/CD
      （E2E 通過後に Vercel へ自動デプロイ、失敗時はマージブロック）

【機能ブロック一覧】
  ① ダッシュボード表示
  ② 在庫管理（入出庫履歴の記録・閲覧、需要予測／発注推奨アラート）
  ③ 注文管理（サマリ、日別売上チャート、商品別ランキング、期間比較、PDF/Excel）
  ④ レポート・分析（売上の分析・比較・エクスポート）
  ＜ユーザー権限管理＞ 管理者ダッシュボード、監査ログ、RLS の細分化（管理者／一般）
  ⑤ 設定（アカウント、DB 接続状況、権限、ダミーテストデータ追加）
  ⑥ 管理者（ユーザー一覧・権限管理）
  ⑦ エラー監視・ログ（Sentry、Vercel Analytics）

--------------------------------------------------------------------------------
2. 機能詳細（①〜⑦）
--------------------------------------------------------------------------------

【① ダッシュボード】
  - 売上推移、期間売上、期間注文数、平均注文額
  - 売上ランキング上位 TOP10
  - 在庫アラート（数量が少ない商品）

【② 在庫管理】
  - 商品グループ数・総 SKU 数・総在庫数・在庫アラートの表示
  - 同一商品の色・サイズ違いは入れ子（階層）表示
  - 商品名・商品コード・JAN で検索
  - CSV テンプレート DL → 入力 → アップロードで商品情報を更新
  - 画面上の在庫数量変更を DB に反映
  - JAN バーコードスキャンで在庫加算し DB を更新

【③ 注文管理】
  - 総注文数、本日の注文、総売上（税込）、平均注文数 等
  - 注文番号クリックで詳細（税込・税別）
  - スキャンで明細に商品追加、数量は ＋／－
  - 合計金額表示、注文確定で一覧・DB に反映

【④ レポート・分析】
  - 売上データの分析・比較、PDF / Excel 出力

【⑤ 設定】
  - アカウント、DB 接続状況、権限、ダミーテストデータ追加

【⑥ 管理者】
  - ユーザー一覧・権限管理

【⑦ エラー監視・ログ】
  - Sentry によるエラー収集
  - Vercel Analytics によるアクセス解析

4. データ・画像・開発の進め方（メモ）
--------------------------------------------------------------------------------

  - 最初は Docker で DB を簡単に立て、あとから移行する案もあり（詳細は要件・技術仕様書）。
  - ダミーデータ参考: doc/template_DB.ods（※メモにあった「template_DB.odsDOC」は doc フォルダ内の ODS を指す想定）
  - ODS 内の画像指定: 商品ごとに画像を載せたい → Storage 等への格納＋ URL／パスをマスタに持つ案が現行設計に近い（要件・機能設計書参照）。
